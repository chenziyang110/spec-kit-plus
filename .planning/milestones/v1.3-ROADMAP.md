# Roadmap: spec-kit-plus

## Overview

The roadmap for milestone v1.3 focused on turning `sp-implement` into a truthful leader/worker runtime. The milestone established the leader-only scheduler contract, made delegated worker dispatch and mixed failure handling real, and then surfaced that runtime truth across planning artifacts, generated surfaces, release-facing docs, and focused verification.

## v1.3 Implement Orchestrator Runtime

**Milestone Goal:** Make `/sp.implement` a milestone-level orchestration leader that delegates all concrete execution work to workers and can continue through the full active roadmap until completion or a clear blocker.

## Milestone Summary

**Key Decisions:**

- Keep the shared `implement` template as the primary source of truth for runtime wording.
- Preserve roadmap order as the default execution contract while allowing only narrow safe preparation work.
- Use mixed failure handling so non-critical failures do not freeze unrelated work while critical failures surface blockers truthfully.

**Issues Resolved:**

- The milestone-level loop no longer stops at a single phase boundary.
- Worker batch convergence, retry taxonomy, and blocker handling are now encoded in runtime state instead of implied in chat.
- Shared template wording, generated Codex output, and release-facing docs now describe the same runtime story.

**Issues Deferred:**

- Applying the same leader/worker model to `debug`
- Broader integration rollout beyond the current Codex-centric runtime slice
- More durable coordination substrate than the current file-backed runtime state

**Technical Debt Incurred:**

- No milestone-level tech debt blockers were recorded in the milestone audit.
