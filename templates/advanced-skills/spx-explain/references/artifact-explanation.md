# Artifact explanation

When branch context is ambiguous, resolve the lane with
`{{specify-subcmd:specify-runtime lane resolve --command explain}}`.

Choose evidence by question:

- execution: task index/tasks, then plan and referenced specification;
- architecture: plan contract, named research/contracts, then live owners;
- requirements: spec contract/spec and referenced discussion or UI evidence;
- current repository behavior: source, tests, configuration, and runtime output;
- cognition: status/Compass output plus the live files behind the claim;
- compatibility/export: the named atlas or handbook view and its source files.

If an expected artifact is absent or stale, name the missing layer and reduce
confidence. Do not fill the gap from model memory or silently treat planning
intent as implemented behavior.
