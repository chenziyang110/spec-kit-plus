# UI intake

Read this only when visual or UI references affect the feature. Confirmed
discussion state is governed by `discussion-handoff.md`.

For UI references, record:

- source paths or URLs and requested fidelity: approximate, high, or
  inspiration;
- layout, hierarchy, interaction, responsive, accessibility, and required state
  constraints that are actually visible or confirmed;
- existing design-system tokens/components to adopt;
- states requiring later visual comparison or human review.

Do not infer hidden behavior from a screenshot. Do not reduce original visual
inputs to prose when downstream implementation needs to inspect them. Use
`assets/ui-brief.md` only when the UI contract adds decision value; otherwise
keep the feature contract compact.
