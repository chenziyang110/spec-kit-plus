# Constitution contract

Principles are durable project constraints, not feature requirements or coding
style preferences. Include only rules that should govern repeated downstream
decisions.

For every material amendment, capture:

- normative obligation and affected scope;
- rationale and conflict precedence;
- evidence or project decision supporting it;
- downstream artifact classes that may require revalidation;
- amendment/version note required by the existing constitution format.

Prefer testable wording such as required review, validation, compatibility, or
ownership behavior. Avoid universal language that the repository cannot honor.
Changing a principle does not retroactively make existing artifacts compliant;
surface that debt as a follow-up route.
